1.背景介绍

json因为其高效,易读的原因而成为前后端交互数据的主要方式.而我们在项目中都是通过json-taglib标签实现从java的各种对象到json的转换.

json-taglib最早是在2007年2月发布第一版,最近一次发布是在2008年1月,是一个比较成熟的技术

那么为什么我们要在项目中使用json-taglib来获取json数据呢?

The JSON taglib has been designed to be as easy to use as possible. It does only one thing - renders JSON data from within a JSP.

上面这段话是Json-taglib官网主页上的一句话,它实际上已经给出了刚才那个问题的答案.它的意思就是该标签被设计的初衷就是使用简单.它只做一件事,那就是从jsp中提取json数据.

2.知识剖析

如何使用json-taglib标签。

首先,要下载jar包到你的项目中.该标签的jar包在maven仓库中没有,我们需要手动将它添加进项目中.

其次,在jsp页面中声明标签

"<%@ taglib prefix="json" uri="http://www.atg.com/taglibs/json" %>"

该标签包含了三个简单的标签,json:object,表示json数据对象;json:property,表示json对象中的属性;json:array,表示json数组,对象中可以有数组,数组中也可以有对象,可以使用这三个标签的嵌套生成各种格式的json数据,完全可以满足我们开发的需求

这里可以看一下官网给出的示例

4.解决方案

写一个demo,来实际感受一下几种方式的区别

在代码实战之前,先了解一下如果不用json-taglib,那么我们使用哪种方式向前端传递json数据呢?

通过@ResponseBody注解的方式

如果在控制器方法上标注该注解,那么方法返回的对象不会经过视图解析器的渲染,而是根据HTTP Request Header的Accept字段的内容,转换为指定的格式,写入到Response对象的body数据区中,如果没有额外的配置,那么就会返回json格式的数据

这段话是是从参考文献三中直接拷贝过来的

5.实战

1.使用json-taglib,在控制器中的代码更加简洁,易读

2.使用json-taglib更加灵活,如果以后需要更改json数据格式,只需要更改jsp页面即可,不需要改动控制器代码

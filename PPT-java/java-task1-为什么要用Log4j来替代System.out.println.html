1.背景介绍

log4j也就是记录日志，可以作为日后处理问题的依据，方便开发者根据日志来统计查询处理问题。此外，查阅日志内容了解程序的运行状况，发现项目中存在的隐患

2.知识剖析

Lof4j是什么

1.Log4j是Apache的一个开放源代码项目，通过使用Log4j，我们可以控制日志信息输送目的地是控制台、文件、GUI组件、甚至是套接口服务器、NT的事件记录器、UNIX Syslog守护进程等；可以轻松的控制log信息是否显示、输出方式、输出格式，更加细致地控制日志的生成过程。而其通过配置文件可以灵活地进行配置而不需要大量的更改代码。可以通过在代码中嵌入许多打印语句，也可以构造一个日志操作类来封装此类操作我们也可以控制每一条日志的输出格式

log4j由三个重要组件构成

1.日志信息的优先级

2.日志信息的输出目的地

3.日志信息的输出格式

日志输出的优先级

log4j.rootLogger=[level],appenderName,appenderName,

官方推荐使用的level有四种，从低到高分别是debug，info,warn,error

appenderName是目的地的参数名，可以同时指定多个输出目的地

目的地（控制台、文件等等）

log4j.appender.appenderName=fully.qualified.name.of.appender.class

log4j提供了5种输出目的地：1.org.apache.log4j.ConsoleAppender(控制台）2.org.apache.log4j.FileAppender(文件）3.org.apache.log4j.DailyRollingFileAppender(每天产生一个日志文件）4.arg.apache.log4j.RollingFileAppender(文件大小到达指定尺寸的时候产生一个新的文件）5.org.apache.log4j.WriterAppender(将日志信息以流格式发送到任意指定的地方）

配置日志信息的输出格式

log4j.appender.appenderName.layout=fully.qualified.name.of.layout.class

log4j提供了四种格式：1.org.apache.log4j.HTMLLayout(以HTML表格形式布局）2.org.apache.log4j.PatternLayout(可以灵活地指定布局模式）3.org.apache.log4j.SimpleLayout(包含日志信息的级别和信息字符串）4.arg.apache.log4j.TTCCLayout(包含日志产生的时间、线程、类别等等信息）

3.常见问题

1.log4j有什么作用

2.为什么要用log4j来替代System.out.println

4.解决方案

1.监视代码中变量的变化情况

2.周期性的记录到文件中供其他应用进行统计分析工作

3.跟踪代码运行时轨迹，作为日后审计的依据

4.担当集成开发环境中的调试器的作用，向文件或控制台打印代码的调试信息

log4j 在脱离ide的情况下，如果不想用system.out.println的话，就只能一行行注释掉，如果再想用的话，再一行行恢复，特别麻烦。如果用log4j，相比起system.out方便的多。并且system.out只能输出到控制台，而log4j之类的日志工具可以配置输出目的地，减少输出形式与程序的耦合（对象之间的依赖）。

1.背景介绍

Struts2和SpringMVC都是支持MVC结构的框架,也是JAVA后端开发中的最常用的两个框架.

两者完成的功能都是负责处理访问服务器的请求,都对传统的servlet请求进行封装,简化开发流程.

Struts2框架在2007年发布第一版,而SpringMVC作为Spring框架的一个子项目,时间上要比Struts2框架更晚一些.Struts2框架在当年可以说是绝对的主流,但是这两年Struts框架日渐式微,有被SpringMVC框架逐渐取代的趋势,不过瘦死的骆驼比马大,现在还是有很多公司在使用这个框架,许多后端工程师的招聘需求上也要求要会使用Struts2框架

那么这两个框架的区别是什么?

2.知识剖析

Struts2和SpringMVC框架之间的区别

1.核心控制器: 即用来处理请求的核心组件,SpringMVC的核心控制器是servlet,Struts2的核心控制器是Filter.

2.控制器实例：Spring Mvc是基于方法设计，而Sturts是基于对象，每次发一次请求都会实例一个action，每个action都会被注入 属性，而SpringMVC更像Servlet一样，只有一个实例，每次请求执行对应的方法即可(注意：由于是单例实例，所以应当避免全局变量的修改，否则会产生线程安全问题)

3.管理方式: 两者都可以使用spring来管理,而两者在管理方式上的主要区别是对注解的支持程度,因为spring mvc是spring中的一个模块，所以spring对于spring mvc的控制器管理更加简单方便一些.

4.参数传递: Struts2中自身提供多种参数接受，其实都是通过（ValueStack）进行传递和赋值，而SpringMvc是通过方法的参数进行接收

5.interceptor的实现机制: Struts2有以自己的interceptor机制，Spring mvc用的是独立的AOP方式。所以struts的配置文件量还是比spring mvc大.

以上5点资料来源于: SpringMvc和Struts2的对比

为什么SpringMVC框架会逐步取代Struts2框架?

1. 使用简单,学习成本低.从搭建Struts2框架的经历来看,SpringMVC的学习难度小于Struts2,Struts2框架搭建起来要更加复杂

2. SpringMVC很容易就写出性能优秀的程序,但是Struts2则要处处小心才能写出性能优秀的程序

3. SpringMVC对Ajax的支持更加优秀.因为SpringMVC集成了Ajax,只需要添加@ResponseBody注解或者通过jsp就可以实现返回json数据,而Struts2要传递json数据,就必须要安装插件或者自己集成,使用不方便

4. SpringMVC对restful风格接口的支持更加优秀.这一点我们做任务的时候就可以看出来,只需要将@RequestMapping中的参数设置成restful风格的接口,设置相应的HTTP方法即可,但是Struts2如果实现restful风格的接口,还需要我们添加rest风格的struts插件,修改配置文件,修改源码才可以,非常繁琐

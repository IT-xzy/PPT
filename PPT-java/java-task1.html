1.背景介绍

在数据库中对ID字段应用自增会出现一些问题，也就表明了某些场景下是不推荐自增ID的

2.知识剖析

什么是自增ID

自增ID在MySQL数据库中，可通过sql语句AUTO_INCREMENT来对特定的字段启用自增赋值 使用自增ID作为主键，能够保证字段的原子性.

自增ID的好处

1.在添加一行时无需对自增id列赋值
2.单个表中能够唯一标识该字段，保证原子性

自增ID的坏处

1.不存在连续性

2.想要手动插入/更新指定id很繁琐

3.在面向对象时无法保证完整性

4.合并表会出现ID重复的情况

什么场景下不使用自增ID

1.其他字段能唯一标识该行数据时无需自增ID

2.分布式数据库不可使用自增ID

3.需要在一定范围内保证ID唯一

3.常见问题

1.多表合并时id会出现重复的情况

2.分布式数据库环境下无法保证唯一性

4.解决方案

UUID替代自增ID

什么是UUID

UUID含义是通用唯一识别码 (Universally Unique Identifier)，指在一台机器上生成的数字，它保证对在同一时空中的所有机器都是唯一的。通常平台会提供生成的API。

UUID存在的目的

UUID的目的，是让分布式系统中的所有元素，都能有唯一的辨识信息，而不需要通过中央控制端来做辨识信息的指定。如此一来，每个人都可以创建不与其它人冲突的UUID。在这样的情况下，就不需考虑数据库创建时的名称重复问题。
